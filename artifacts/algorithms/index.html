Algorithms and Data Structures Artifact

Overview

The Course Planner application is a C++ program developed in CS-300 Data Structures and Algorithms. The program loads course information from a CSV file, stores course data, allows users to search for specific courses, displays prerequisites, and prints course information in alphanumeric order.

This artifact is being used for the Algorithms and Data Structures enhancement category in the CS-499 Computer Science Capstone.

Features

  • Load course data from a CSV file
  • Store course information using vectors
  • Search for courses by course number
  • Display prerequisite information
  • Print course lists in alphanumeric order
  • Analyze efficiency and tradeoffs between data structures

Algorithms and Data Structures

This project focuses on core data structure and algorithm concepts, including:

  • Vectors
  • Searching algorithms
  • Sorting algorithms
  • Runtime complexity analysis
  • Binary search trees
  • Hash tables
  • Data structure performance tradeoffs

The original implementation stores and retrieves course data using vectors while analyzing alternative data structures such as hash tables and binary search trees for improved performance and efficiency.

Enhancement Completed

The enhancement for this category improved the Course Planner by expanding the original vector-based design with an unordered_map hash table for faster course lookup while keeping the vector for sorted course output. The completed changes also include data-loaded validation, CSV row validation, duplicate course detection, case-insensitive course searches, improved menu input handling, and clearer comments explaining the new logic.

Running the Project

This artifact is a C++ console application. It does not require Node.js, Angular, MongoDB, or Postman. The main tool needed is Visual Studio with C++ desktop development support.

Applications and Tools Needed

  • Visual Studio 2022 - Used to open, build, and run the C++ solution.
  • C++ Desktop Development Workload - Required in Visual Studio to compile the project.
  • CSV Input File - The file CS 300 ABCU_Advising_Program_Input.csv must remain in the project directory.
  1. Open CoursePlanner.sln in Visual Studio 2022.
  2. Build the solution.
  3. Run the program using Ctrl + F5.
  4. Use the menu to load course data, print the course list, search for a course, or exit.

The included CSV file must remain in the project directory so the program can load course information correctly.

Testing and Validation

The following tests were used to verify the Algorithms and Data Structures enhancement for the CS-300 Course Planner program. These tests confirm that the program loads data, validates input, prints courses in order, and uses the enhanced hash table lookup correctly.

1. Run the Program

  1. Open CoursePlanner.sln in Visual Studio 2022.
  2. Build the solution.
  3. Run the program using Ctrl + F5.

Expected Result: The menu should display options to load data, print the course list, print a course, and exit.

2. Data Not Loaded Validation Test

  1. Run the program.
  2. Before loading the CSV file, choose option 2 to print the course list.
  3. Before loading the CSV file, choose option 3 to search for a course.

Expected Result: The program should tell the user to load the course data first.

3. Load Course Data Test

  1. Select option 1.
  2. Press Enter to use the default CSV file name, or type CS 300 ABCU_Advising_Program_Input.csv.
  3. Confirm that the program reports the course data loaded successfully.

Expected Result: The CSV file should load without errors.

4. Print Course List Test

  1. After loading the CSV file, select option 2.
  2. Review the printed course list.

Expected Result: Courses should print in alphanumeric order by course number.

5. Valid Course Search Test

  1. After loading the CSV file, select option 3.
  2. Search for CSCI300.

Expected Result: The program should display the course title and prerequisites for CSCI300.

6. Case-Insensitive Search Test

  1. Select option 3.
  2. Search for csci300 using lowercase letters.

Expected Result: The program should still find the course because input is converted to uppercase before searching.

7. Invalid Course Search Test

  1. Select option 3.
  2. Search for TEST999.

Expected Result: The program should display Course not found.

8. Menu Input Validation Test

  1. At the main menu, type a non-numeric value such as abc.
  2. Then type an invalid numeric option such as 7.
  3. Then type a valid option such as 1, 2, 3, or 9.

Expected Result: Non-numeric input should show an invalid input message, invalid menu numbers should be rejected, and valid menu options should still work.

9. CSV Validation Test

  1. For testing only, add invalid rows to a copy of the CSV file, such as:
    • BADCOURSE
    • ,Missing Course Number
    • CSCI300,Duplicate Algorithms
  2. Run the program and load the test CSV file.

Expected Result: The program should skip incomplete or duplicate course records and continue loading valid records.

10. Hash Table Lookup Verification

  1. Load the course data.
  2. Search for a valid course by course number.
  3. Confirm that the search returns the correct course without looping through the vector.

Expected Result: The enhanced program should use the unordered_map hash table for faster course lookup while keeping the vector for sorted course printing.

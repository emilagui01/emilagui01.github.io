Database Artifact

Overview

Travlr is a full-stack travel booking web application developed using the MEAN stack: MongoDB, Express, Angular, and Node.js. For the database category, this artifact focuses on the MongoDB and Mongoose portions of the application, including schemas, CRUD operations, indexed fields, validation, and database-driven application behavior.

This artifact was originally developed in CS-465 Full Stack Development I and is being used for the Database enhancement category of the CS-499 Computer Science Capstone.

Database Features

  • MongoDB database integration
  • Mongoose schema modeling
  • CRUD operations for trip management
  • RESTful API communication with database-backed routes
  • JWT-protected API endpoints
  • Indexed database fields
  • Validation for structured trip data

Project Structure

  • app_api/models/ - Mongoose schema definitions
  • app_api/controllers/ - Backend logic for database operations
  • app_api/routes/ - API endpoints connected to MongoDB operations
  • data/ - Seed and supporting data files
  • app_admin/ - Angular frontend that communicates with database-backed API routes

Running the Project

Since generated dependency folders such as node_modules were removed to reduce file size, dependencies must be reinstalled before running the project.

Install backend dependencies:

npm install

Install Angular dependencies:

cd app_admin
npm install

Start the backend application:

npm start

Start the Angular development server:

cd app_admin
ng serve

Testing and Validation

Purpose: Verify that MongoDB, Mongoose validation, controller error handling, and JWT-protected database access work correctly.

  1. Start MongoDB, the backend server, and the Angular admin application.
  2. Use Postman to send a valid POST /api/register request and confirm a token is returned.
  3. Send the same registration request again and confirm the API returns 409 Conflict for duplicate email.
  4. Send an invalid email during registration and confirm the API returns 400 Bad Request.
  5. Use Postman to send a valid POST /api/trips request with a Bearer token and confirm the trip is created.
  6. Send invalid trip data and confirm the API returns 400 Bad Request instead of crashing the backend.
  7. Use PUT /api/trips/:tripCode with valid data and confirm the trip updates successfully.
  8. Use invalid update data and confirm MongoDB does not overwrite the valid record.
  9. Remove the Bearer token and confirm protected write routes return 401 Unauthorized.

Software Engineering and Design Artifact

Overview

Travlr is a full-stack travel booking web application developed using the MEAN stack: MongoDB, Express, Angular, and Node.js. The application includes an Angular administrative interface, a Node.js and Express backend API, MongoDB database integration, and JWT-based authentication for protected routes.

This artifact was originally developed in CS-465 Full Stack Development I and is being used for the Software Engineering and Design category of the CS-499 Computer Science Capstone.

Features

  • Angular single-page application
  • RESTful API architecture
  • MongoDB database integration with Mongoose
  • JWT authentication and authorization
  • Protected API routes
  • CRUD operations for trip management
  • Reusable Angular components and services

Project Structure

  • app_admin/ - Angular administrative frontend
  • app_api/ - API controllers, models, authentication, and routes
  • app_server/ - Express server-side components
  • public/ - Static assets
  • data/ - Seed and supporting application data

Enhancement Completed

The enhancement for this category improved the design, maintainability, usability, and security of the Travlr administrator application. The completed changes include an Angular route guard for protected admin pages, improved login flow, stronger Add Trip and Edit Trip form validation, clearer user-facing error messages, a read-only trip code field during edits, Delete Trip functionality, reusable service updates, and simple comments explaining important TypeScript logic.

Running the Project

This artifact can be tested through the Travlr administrator web application. The main tools needed are Visual Studio Code, Node.js/npm, Angular CLI, and a web browser. Postman is not required for this software design test because these tests focus on the user interface, route protection, form validation, login flow, and admin CRUD features.

Applications and Tools Needed

  • Visual Studio Code - Used to open and review the project files.
  • Node.js and npm - Used to install dependencies and run the backend server.
  • Angular CLI - Used to run the Angular administrator application.
  • Web Browser - Used to test the Travlr admin interface at http://localhost:4200.

Since generated dependency folders such as node_modules were removed to reduce file size, dependencies must be reinstalled before running the project.

Install backend dependencies from the main Travlr project folder:

npm install

Install Angular dependencies:

cd app_admin
npm install

Start the backend application from the main Travlr project folder:

npm start

Start the Angular development server in a separate terminal:

cd app_admin
ng serve

After both servers are running, open the admin application in a browser: 

http://localhost:4200

Testing and Validation

The following tests were used to verify the Software Engineering and Design enhancement for the Travlr administrator application. These tests confirm that route protection, login flow, validation, CRUD functionality, and user-facing error handling work correctly.

1. Route Guard Protection Test

  1. Start the backend server with npm start.
  2. Start the Angular admin application with cd app_admin and ng serve.
  3. Open the application at http://localhost:4200.
  4. Make sure the user is logged out.
  5. Manually type http://localhost:4200/add-trip into the browser.
  6. Confirm that the application redirects to the login page.
  7. Repeat the test with http://localhost:4200/edit-trip.

Expected Result: Logged-out users should not be able to access protected admin routes directly.

2. Login Flow Test

  1. Navigate to the Login page.
  2. Enter valid login credentials.
  3. Submit the form.
  4. Confirm that the JWT token is saved and the user is redirected back to the trip listing page.
  5. Log out and try logging in with invalid credentials.

Expected Result: Valid login should redirect to the trip listing page. Invalid login should display a user-facing error message.

3. Add Trip Validation Test

  1. Log in as an authenticated user.
  2. Go to the Add Trip page.
  3. Enter invalid values such as:
    • ab for Code
    • hi for Name
    • x for Length
    • blank Start Date
    • aa for Resort
    • abc for Per Person
    • reef1 for Image Name
    • short for Description
  4. Click Save.

Expected Result: The form should display validation messages and prevent invalid trip data from being submitted.

4. Add Trip Valid Submission Test

  1. Go to the Add Trip page.
  2. Enter valid trip data:
    • Code: TEST2026
    • Name: Test Trip
    • Length: 4 nights
    • Start: 2026-05-07
    • Resort: Test Resort
    • Per Person: 799.99
    • Image Name: reef1.jpg
    • Description: This is a test trip description.
  3. Click Save.
  4. Confirm that the trip appears in the trip listing.

Expected Result: Valid trip data should save successfully and appear on the trip listing page.

5. Edit Trip Test

  1. Log in as an authenticated user.
  2. Click Edit Trip on an existing trip.
  3. Confirm that the Code field is read-only.
  4. Update fields such as Name, Length, Resort, Price, Image, or Description.
  5. Click Save.
  6. Confirm that the trip updates successfully on the trip listing page.

Expected Result: The trip should update successfully, and the trip code should not be accidentally changed.

6. Delete Trip Test

  1. Log in as an authenticated user.
  2. Create a test trip or choose an existing test trip.
  3. Click Delete Trip.
  4. Confirm the browser confirmation popup appears.
  5. Click OK.
  6. Confirm the trip is removed from the trip listing.

Expected Result: The selected trip should be deleted only after confirmation.

7. Error Handling Test

  1. Stop the backend server while the Angular frontend is still running.
  2. Try to add or edit a trip.
  3. Observe the error message shown in the interface.

Expected Result: The application should display a clear error message instead of only logging the error in the browser console.

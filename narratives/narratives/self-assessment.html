Professional Self-Assessment

Throughout the Computer Science program, I have developed a stronger understanding of how software systems are planned, designed, developed, tested, secured, and improved over time. When I first started the program, my main goal was to build a solid foundation in programming and software development. As I moved through more advanced courses, my career interests became more focused on cybersecurity because I saw how closely security connects to software design, networking, databases, authentication, and secure coding. Completing this ePortfolio helped me connect the different parts of my coursework into one professional collection that shows my growth as a computer science student and my preparation for future technology roles. After graduation, my goal is to pursue IT internship opportunities and continue building my qualifications through certifications such as Network+ and Security+ as I work toward a cybersecurity career.

One of the most important strengths I developed throughout the program is my ability to design and improve software systems. Earlier programming courses helped me practice core development skills, while later courses pushed me to think more carefully about structure, maintainability, user needs, and security. In the Travlr Getaways full-stack application, I worked with Angular, Express.js, Node.js, MongoDB, RESTful API routes, authentication, and reusable components. Enhancing this artifact helped me move beyond simply making the application function. I had to evaluate how the application handled access control, user input, error handling, database operations, and administrative workflows. By improving route protection, login behavior, form validation, CRUD functionality, database validation, and JWT-protected backend access, I strengthened my ability to evaluate an existing system and improve it in a way that is more reliable and professional.

My coursework also helped me develop stronger skills in data structures and algorithms. The CS-300 Course Planner artifact gave me experience working with file input, course records, prerequisite relationships, vectors, sorting, searching, and hash-based lookup. The original program used a vector to store and search course data, which was understandable but not the most efficient option for course lookup. During the enhancement, I kept the vector for sorted course printing and added an unordered_map hash table for faster course searching. This helped me understand that choosing the right data structure depends on the purpose of the operation. A vector can be useful for ordered display, while a hash table is better for fast lookup by course number. This project also strengthened my ability to handle invalid input, validate file data, avoid duplicate records, and improve the reliability of a console-based application.

Database development became another important part of my growth. In the Travlr application, I worked with MongoDB and Mongoose to store, retrieve, update, and delete trip and user records. For the database enhancement, I improved the Mongoose schemas, changed the perPerson field from a string to a number, added stronger validation rules, improved controller error handling, used findOne() for single-trip lookup, and added validation during update operations. I also improved user registration by validating email format and handling duplicate email addresses more clearly. These changes helped me better understand that database work is not only about storing information. It also involves protecting data integrity, choosing appropriate field types, validating data before it is saved, and making sure backend errors are handled safely. Testing the database changes with Postman helped me confirm that valid records were saved, invalid records were rejected, and protected operations required a valid JWT token.

Security has become one of the main areas I want to continue developing professionally. In CS-305 Software Security, I worked with secure coding practices, cryptographic hashing, HTTPS configuration, and vulnerability assessment tools. That project helped me understand that secure development requires more than adding a security feature at the end. It requires identifying risks, using trusted tools, testing for vulnerabilities, and making decisions that protect users and organizations. In my capstone work, I applied that same security mindset by improving authentication flow, protecting administrative routes, validating data at both the frontend and database layers, and ensuring protected database operations require valid authorization. These experiences connect directly to my cybersecurity goal because they show how software development, databases, and security all support one another.

Testing and quality assurance were also important parts of my development as a computer science student. In CS-320 Software Test Automation and Quality Assurance, I practiced using unit tests and validation rules to verify that software met requirements. That work helped me understand how tests can turn user expectations into specific conditions that a program must satisfy. It also showed me that testing improves both functionality and security because it helps identify invalid inputs, edge cases, and logic errors before they become larger problems. I carried that mindset into my capstone enhancements by testing valid and invalid form input, database requests, update operations, duplicate records, and protected routes. This process helped me become more careful about confirming that changes worked as intended instead of assuming that a successful code change automatically meant the system was reliable.

Another area of growth was communication with stakeholders and translating user needs into technical designs. In CS-255 System Analysis and Design, I worked on the DriverPass system, which required analyzing client needs and turning them into business requirements, technical requirements, UML diagrams, and system design decisions. That project helped me understand that software development begins before coding. Developers need to understand what users and stakeholders actually need, document those needs clearly, and design systems that solve the right problems. This experience also helped me communicate technical ideas in a more organized way. The ability to explain design decisions is important because computer science work often involves collaboration with instructors, peers, managers, clients, or nontechnical stakeholders who need to understand why a system is being built a certain way.

Developing this ePortfolio also helped me improve my professional communication. The code review video required me to explain existing code, identify weaknesses, and discuss planned improvements in a way that could be understood by an instructor or peer reviewer. The artifact narratives required me to explain not only what I changed, but why those changes mattered. Through this process, I became better at connecting technical work to course outcomes and professional skills. Instead of only saying that I know Angular, C++, MongoDB, or secure coding, the ePortfolio gives specific evidence of how I used those skills to improve real projects. This is especially important as I prepare to apply for IT internships and continue building toward cybersecurity roles, because employers need to see both technical ability and clear communication.

The artifacts in my ePortfolio fit together by showing different but connected areas of computer science. The Travlr Software Design and Engineering enhancement demonstrates my ability to improve a full-stack web application, strengthen user experience, add CRUD functionality, protect routes, and organize reusable components. The CS-300 Course Planner Algorithms and Data Structure enhancement demonstrates my ability to evaluate data structure tradeoffs, improve search efficiency, sort and search course records, and make a program more reliable through input validation. The Travlr Databases enhancement demonstrates my ability to improve MongoDB and Mongoose functionality, validate stored data, protect database write operations, and handle backend errors more safely. Together, these artifacts show that I can work across frontend development, backend development, databases, algorithms, testing, and security.

This portfolio also reflects the professional values I want to carry forward. I want to build systems that are reliable, maintainable, secure, and useful to the people who depend on them. My long-term goal is to move into cybersecurity, but this capstone helped me see that cybersecurity is connected to many parts of computer science. Secure systems require good software design, strong data validation, reliable databases, careful testing, and clear communication. As I move toward graduation, IT internships, and certifications, I plan to keep building on the skills shown in this ePortfolio. Overall, completing the Computer Science program and developing this capstone portfolio helped me better understand my strengths, clarify my career direction, and prepare to continue growing as a technology professional.
